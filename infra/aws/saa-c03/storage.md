**EC2 Instance Store**

EC2 가 연결된 물리적인 서버의 하드웨어 드라이브를 가리킵니다.

훌륭한 처리량을 갖추고 있어서 매우 향상된 디스크 성능이 필요할 때 I/O 성능 향상을 위해 활용할 수 있습니다.

하지만 EC2 Instance 를 종료하면 삭제되기 때문에 장기적으로 데이터를 보관할 수는 없습니다.
장기적인 데이터 보관에는 EBS 가 적합합니다.

따라서 버퍼나 캐시, 스크래치 데이터 또는 임시 콘텐츠 사용 시에 사용됩니다.

**EBS(Elastic Block Store)**

인스턴스가 실행 중인 동안 연결 가능한 네트워크 드라이브(네트워크 USB)
CCP 레벨 : 하나의 EBS 는 하나의 EC2 인스턴스에만 마운트 가능합니다.

어소시에이트 레벨 :

io1 / io2 유형의 EBS에 해당 가용 영역 내의 16개의 EC2 인스턴스까지 다중 연결합니다.

1. EBS 볼륨을 사용하면 인스턴스가 종료된 후에도 데이터를 지속할 수 있습니다.
   인스턴스를 재생성하고 이전 EBS 볼륨을 마운트함으로서 데이터를 지속할 수 있는 것입니다.
   **EC2 인스턴스 종료시 삭제 기능도 존재합니다.**
   EC2 인스턴스에서 EBS 볼륨을 분리할 필요는 없지만 권장 사항입니다.

2. EBS 는 특정 Availavbility Zone(AZ) 에 종속되어 있습니다.
   us-east-1a 에 생성된 EBS 는 use-east-1b 에서 연결할 수 없습니다.
   하지만 스냅샷을 이용해서 다른 리전이나 AZ 로 이동이 가능합니다.

유형
크기, 처리량 및 초당 입출력 작업 수로 분류됩니다.
EC2 의 부팅 볼륨으로 gp2, gp3, io1, io2 만 사용할 수 있습니다.

1. gp2 / gp3(SSD) : 다양한 워크로드에 대해 가격과 성능의 균형을 맞추는 범용(General Purpose) SSD 볼륨입니다.
2. io1 / io2(SSD) : 가장 성능이 좋습니다. 미션 크리티컬, 저지연, 고처리량 작업에 사용됩니다.
3. st1 (HDD) : 저비용 대용량 볼륨으로 자주 액세스하고 처리량이 많은 작업에 사용됩니다.
4. sc1 (HDD) : 가장 저렴한 HDD 볼륨으로 액세스 빈도가 낮은 작업에 사용됩니다.

**EBS Snapshots**

EBS 볼륨의 특정 시점에 대한 백업입니다.
스냅샷을 이용해서 다른 리전이나 AZ 로 이동이 가능합니다.

특징

1. EBS Snapshot Archive

   최대 75%까지 저렴한 아카이브 티어 스냅샷을 옮길 수 있습니다.

   아카이브를 복원하는데 24시간에서 최대 72시간이 걸립니다.

2. EBS Snapshot Recycle Bin
   EBS 스냅샷을 삭제하는 경우 영구 삭제하는 대신에 휴지통에 넣을 수 있습니다.
   실수로 삭제하는 경우에 휴지통에서 복원 가능하며 보관기간은 1일에서 1년 사이로 설정 가능합니다.
3. Fast Snapshot Restore(FSR)
   스냅샷을 완전 초기화해 첫 사용에서의 지연 시간을 없애는 기능입니다.
   스냅샷이 아주 크고 EBS 볼륨 또는 EC2 인스턴스를 빠르게 초기화해야 할 때 유용합니다.
   하지만 비용이 많이 듭니다.

**EFS(Elastic File System)**

관리형 NFS(Network File System) 입니다.

많은 EC2 Instance 에 마운트 가능합니다.

이 때 EC2 Instance 는 서로 다른 AZ에 존재해도 가능합니다.(Regional 설정과 One Zone 설정으로 분리)

Linux 기반의 AMI 와만 호환됩니다.

사용량(GB) 에 따라 자동으로 비용이 발생하기 때문에 프로비저닝이 필요하지 않습니다.

콘텐츠 관리, 웹 서빙, 데이터공유, Wordpress 에 사용됩니다.
